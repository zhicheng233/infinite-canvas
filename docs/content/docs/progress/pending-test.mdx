# 待测试确认

- [ ] 登录 / 注册 / 个人中心流程在当前商用部署中可正常使用。
- [ ] 空数据库首次启动且已配置 `INIT_ADMIN_*` 时，会自动创建初始管理员。
- [ ] 未登录访问首页显示落地页，登录后进入业务工作区。
- [ ] 未登录首页展示更完整的商业化落地页，包括能力、场景、积分和后台统一配置说明。
- [ ] 普通用户不可见管理后台入口，也不可访问管理接口。
- [ ] 管理员在 `/admin/api-config` 可为每个视频模型单独指定接口路由。
- [ ] 视频模型未显式指定路由时默认走 `/v1/videos`，不再仅凭模型名自动切到 `xai` 或 `newapi` 分支。
- [ ] 管理员在 `/admin/model-logs` 可查看模型失败日志，并按模型、用户、类型和关键词筛选。
- [ ] 视频模型路由选择 `yijia` 后，应按 `/v1/videos` JSON 格式提交 URL `input_reference`、`seconds`、`watermark` 等字段。
- [ ] 管理员在 `/admin/api-config` 可为每个视频模型单独配置可选时长；前台视频时长面板应只显示后台允许的选项。
- [ ] 管理员在 `/admin/api-config` 可为每个图片模型分别配置“文生图接口”和“图生图接口”。
- [ ] 旧的 `model_routes.image:<model>` 配置仍会兼容为图生图接口，无需手工改库即可继续使用。
- [ ] Banana 系列图片模型使用独立的 Banana 图片路由，走 `/chat/completions` 且携带 `extra_body.google.image_config`；普通多模态聊天生图路由不携带该参数。
- [ ] 未配置计费的模型不会出现在前台，也无法通过后端代理调用。
- [ ] 当视频任务返回跨域成片地址（如 `https://api.waninter.com/v1/videos/.../content`）时，前端应通过后端代理成功拉取成片并正常展示，不再一直停留在生成中或空白。
- [ ] 画布、图片工作台、视频工作台、音频工作台的生成记录都按账号隔离。
- [ ] 画布数据按账号隔离，A 用户不能看到 B 用户的新画布。
- [ ] 图生图链路改为真正的编辑请求后，参考图修改行为符合预期。
- [ ] 视频工作台已改为走后台统一 API 配置，不再提示用户自行配置 API Key。
- [ ] 用户可在 `/credits` 查看积分明细，在 `/recharge` 查看充值记录。
- [ ] 积分流水应显示变动前后余额、模型、接口路径、计费单位、单价、数量和后台操作人等详情。
- [ ] 图片工作台、视频工作台和画布生成入口在生成前展示预计扣除积分；余额不足时引导充值。
- [ ] 管理员可在 `/admin/api-config` 为视频模型选择“视频动态”计费，并配置基础积分、480p/720p/1080p/2K/4K 秒单价。
- [ ] 视频工作台和画布视频节点在选择不同秒数、分辨率后，应按后台动态计费规则展示预估积分，并在任务创建成功后扣除相同积分。
- [ ] 新服务器按 Docker 文档迁移后，账号、积分、画布、生成记录与后台模型配置可正常恢复。
- [ ] 新服务器使用 `docker compose up -d --build` 部署后，登录接口应命中 `.env` 中配置的 `NEXT_PUBLIC_API_URL`，不再落到 `http://localhost:8080/api`。
- [ ] 正式域名按同域名区分前缀部署后，前端内置接口走 `https://域名/api/*`，Go 后端接口走 `https://域名/backend-api/*`，登录、画布和生成接口可正常访问。
- [ ] 正式环境构建前端时，不应再出现 `"/backend-api/canvas" cannot be parsed as a URL` 这类服务端构建期云同步报错。
- [ ] 提示词中心遇到缺少封面或外链加载失败的条目时，应显示占位卡片，而不是坏图。
- [ ] 全局渠道配置：SuperAdmin 可在 `/admin/api-config` 创建/编辑/启用/禁用渠道，设置上游地址和加密 API Key，触发模型同步。
- [ ] 渠道模型同步：SuperAdmin 同步某渠道后，后端自动从上游 `/v1/models` 拉取模型目录并存入 `channel_models` 表，失败不会丢失已有记录。
- [ ] 按能力独立选渠道：前台图片/视频/文本/音频可按能力分别选择不同渠道，各能力模型列表根据所选渠道独立展示。
- [ ] 渠道模型启停：SuperAdmin 可在渠道内单独启用或禁用模型，被禁模型不计入前台选项，不通过后端代理调用。
- [ ] New-API 指标：SuperAdmin 可配置 New-API 管理端地址，后端按 `hours` 参数（1~720）拉取渠道成功率；前台 `/channel-status` 展示各渠道成功率及模型级指标。
- [ ] 渠道指标页改版：`/channel-status` 支持自定义时间窗口（1/6/12/24/72/168/336/720 小时 + 自定义输入）和渠道折叠展开（默认收起）。
- [ ] 成功率排序：模型选项按渠道成功率排序，无数据模型排在最后；面板区分 `暂无数据` / `未映射` / `已过期` / `获取失败`。
- [ ] 渠道选择持久化：前台各能力渠道选择保存在 localStorage，刷新后恢复；渠道被禁用时自动回退。
- [ ] 同一模型在不同渠道上的路由可独立配置，相同模型名不会因渠道差异产生路由冲突。
- [ ] 渠道选择器新增"Auto（自动路由）"选项，选中后自动从所有启用的渠道中按成功率选择最优模型。
- [ ] Auto 渠道在无可选渠道时隐藏，不显示在渠道下拉列表中。
- [ ] Auto 渠道选中后，前端请求携带 `channel_id=0`，后端自动路由到最优渠道。
- [ ] Auto 渠道选中的模型在 API 调用时应正常命中后端自动路由，不报"所选模型已失效"错误。
