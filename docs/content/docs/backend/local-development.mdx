# 本地开发

当前版本采用 `web/ + backend/ + MySQL + Redis` 的组合，本地调试优先使用完整后端链路。

## 1. 启动完整服务

```bash
cp .env.example .env
docker compose build
docker compose up -d
```

默认访问：

```text
前端：http://localhost:3001
后端：http://localhost:18080/api
```

## 2. 初始化管理员

系统默认注册的新用户角色是 `user`。  
如果是全新数据库，建议直接在 `.env` 里配置 `INIT_ADMIN_USERNAME`、`INIT_ADMIN_PASSWORD` 和 `INIT_ADMIN_DISPLAY_NAME`，让后端首次启动时自动创建 `super_admin`。

如果你已经跳过了这一步，仍然可以手动提权：

```bash
docker exec -it ic-mysql mysql -uroot -p
```

```sql
USE infinite_canvas;
UPDATE users SET role = 'super_admin' WHERE username = '你的管理员用户名';
```

## 3. 后台配置上游模型

登录管理员账号后，进入 `/admin/api-config`：

- 配置统一上游 `API Base URL`
- 配置 `API Key`
- 维护模型目录、状态、计费单位、每次积分消耗
- 为视频模型单独指定接口路由

当前视频模型支持的后台路由类型：

- `openai` → `/v1/videos`
- `xai` → `/v1/videos/generations`
- `newapi` → `/v1/video/generations`
- `seedance` → `/contents/generations/tasks`

建议优先为视频模型显式配置路由，不要完全依赖“自动判断”。

## 4. 单独调试前端

如果只需要改前端页面：

```bash
cd web
bun install
bun run dev
```

## 5. 单独调试文档站

```bash
cd docs
bun install
bun run dev
```

## 常见场景

- 改登录、后台、画布、工作台页面：主要看 `web/`
- 改鉴权、积分、模型代理、生成记录：主要看 `backend/`
- 改后台统一模型配置：主要看 `web/src/app/(user)/admin/api-config/` 与 `backend/handler/api_config_handler.go`
- 改文档站内容：主要看 `docs/content/docs/`
